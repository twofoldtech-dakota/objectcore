`knowledge/INDEX.md` is generated by `bun run kb:index` from
`knowledge/entries/*.md`. Never hand-edit it — it is a build artifact, exactly
like `.claude-plugin/marketplace.json`.

`bun run kb:check` (part of `bun run check`) asserts the committed INDEX.md
byte-matches a fresh render and stays within the 200-line / 25KB budget. A
forgotten `kb:index` or a hand-edit fails the gate. Budget overflow is the
deliberate curate/prune signal — the KB's rot mechanism (the index split alone
does not prevent rot).
