## Findings considered and rejected (so they aren't re-audited)

- **C2 — provenance reference dropped on ingest** (`RegistryDbSink` never sets
  `StoredPlugin.provenance`, though the column and field exist): real but
  cosmetic today — provenance is enforced at publish time, and nothing reads the
  stored value yet. Not worth a plan until a feature surfaces "attested versions."
- **"Fix the DB round-trip / manifest field loss"**: investigated and rejected —
  `RegistryDbSink` reconstructs the manifest from only the catalog-entry fields,
  but `deriveCatalog` copies exactly those fields into entries, so the served
  catalog is lossless for everything the contract contains. The round-trip is
  proven by `packages/registry-db/test/store.test.ts:18`. Not a bug.
